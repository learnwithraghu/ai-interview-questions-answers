Question 11

Safe Tool Calling Design

A stakeholder wants an LLM application that can search the web, query an internal database, and send an email. How do you design the tool-calling architecture to route these intents reliably and safely?

The LLM routes — your code executes. Safety depends on what you allow to auto-run.

LLM is the router: it decides which tool to call and what arguments to pass — your code actually runs the function. You're always in control: the model never executes code directly — it returns a JSON decision, you act on it. Classification is the key design decision: get this wrong and your agent sends emails it shouldn't. HITL gate: pause execution, show the user exactly what the agent is about to do, require explicit approval. Argument validation: validate the LLM's returned arguments against the tool schema before running anything. Loop guard: set a max iteration count (e.g. 10 steps) — agents can loop indefinitely if a tool call keeps failing.

Step 3 — validate: LLMs occasionally hallucinate argument names or pass wrong types — catch it before runtime errors. Step 4 — HITL: for send_email, show the to/subject/body and require a click — not just a model decision. Audit log: record every tool call, arguments, and outcome for compliance and debugging — especially for email and payments. Sales AI example: web search and CRM reads run automatically; send_email shows a confirmation modal first. Tool descriptions matter: write precise, unambiguous tool descriptions — the LLM uses them to route, so vague names cause misfires. Separate executors: each tool is an independent function your code controls — never let the LLM run arbitrary shell commands.

Answer

Say it

"I'd define each capability as a function with a strict JSON schema and register them with the tool-calling API. For routing, the model decides which tool to call — I don't hardcode routing logic. The key design decision is safety: for read-only actions like web search or DB queries, I execute automatically. For anything irreversible like sending an email, I always insert a human-in-the-loop confirmation step. I also validate all tool arguments and log every call for auditability."
