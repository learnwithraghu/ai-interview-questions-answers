Question 07

Extracting Valid JSON

You need to extract structured JSON data (like user profiles) from unstructured customer emails. How would you design the prompt and API call to guarantee valid JSON output?

Think in layers, not just prompts.

Visual Explanation

Five layers of guarantee

1
API JSON mode — forces valid JSON at the token level
2
Schema in system prompt — tells the model exactly which fields to return
3
Temperature = 0 — removes randomness from formatting decisions
4
Pydantic validation — treats every LLM response as untrusted input
5
Retry with error — feeds validation failure back to the LLM to fix it
Layer 1: JSON mode is your strongest guarantee — the model literally cannot output invalid JSON. Layer 2: Without an explicit schema, the model guesses field names and hallucination creeps in. Layer 3: Randomness and structured output are enemies — zero temperature means determinism. Layer 4: Pydantic gives you typed, structured exceptions instead of silent runtime crashes. Layer 5: One targeted retry with the specific error fixes the problem the vast majority of the time.

Visual Explanation

The retry loop

Pipeline: every email enters the LLM call, exits as typed JSON, then hits the validator gate. Validation fail: the specific Pydantic error (e.g. "expected int, got str") is appended to the next prompt. One retry: a targeted retry beats a blind retry — the model knows exactly what to fix. Gate: nothing reaches the database until it passes Pydantic — untrusted output never silently corrupts data. Schema drift: update your Pydantic model first; the system prompt schema should mirror it exactly. Production signal: track validation failure rate — a spike means the model or schema drifted.

Answer

Say it

"I'd use a three-layer approach. First, I'd use the API's structured output or JSON mode to force the model to only emit valid JSON. Second, I'd include the exact schema in the system prompt with an example, and set temperature to zero. Third, on the application side, I'd parse the response with Pydantic. If validation fails, I'd automatically feed the error back to the model and ask it to fix the output — usually one retry is enough."
