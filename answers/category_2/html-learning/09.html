Tip 09

/compact in Claude Code

You are an hour into a complex debugging session with Claude Code — many files edited, lots of back-and-forth. Responses are getting less focused and you're approaching the context window limit. How do you keep the session going without losing your work or starting over?

You don't have to restart — there's a smarter way.

Replaces history: the full back-and-forth is swapped for a structured AI-generated summary — meaning preserved, bulk dropped. Preserved: goals, key decisions, edited files, open tasks, and constraints you explicitly stated. Compressed away: exploratory dead ends, repeated context, verbose clarification exchanges. Token savings: a 50k-token session can compress to under 5k — extending what you can do before hitting the hard limit.

Visual Explanation

When to reach for /compact

Before a new subtask
Clear the old trail before starting the next feature
Responses drifting
Context noise building — reset the signal-to-noise ratio
After a big feature ships
Archive decisions, free up window for the next challenge
Before code review
Cleaner context → sharper analysis, fewer distractions
Don't wait: compact proactively — before responses degrade, not after hitting the hard stop. Ctrl+O: inspect the generated summary to confirm nothing critical was dropped before continuing. CLAUDE.md pairing: durable rules (architecture, preferences) belong in CLAUDE.md — /compact handles ephemeral session state. Auto mode: Claude Code triggers compaction automatically at context limit — but manual use gives you more control over timing.

Answer

Say it

"/compact compresses the full conversation history into a structured summary — key decisions, current code state, and open tasks — and injects it as a fresh context window. It extends the session without losing continuity. I use it proactively before starting a new major subtask, or when I notice responses getting less focused. For anything durable like architecture rules, I put it in CLAUDE.md; /compact handles the ephemeral session trail."
