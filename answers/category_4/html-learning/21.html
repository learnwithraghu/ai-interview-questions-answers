Question 21  ·  AI Engineer Interview Bootcamp

When to Skip Frameworks

Explain the role of GenAI orchestration frameworks like LangChain or LlamaIndex. In what scenarios might you choose not to use them and write custom code instead?

Frameworks accelerate prototyping. They also add abstraction layers you'll eventually debug.

Question 21  ·  What frameworks give you

Framework vs custom code

Use a framework
  • Prompt management
  • RAG pipelines
  • Memory abstractions
  • 100+ integrations
"You want to ship fast"
Write custom code
  • Maximal performance
  • Minimal deps
  • Security-critical apps
  • Precise control
"You need to debug everything"
LangChain: widest ecosystem, fastest prototype for RAG and agent flows. API has changed frequently across versions. LlamaIndex: purpose-built for data ingestion and retrieval. Stronger than LangChain for complex indexing workflows. Framework tax: every abstraction layer is something you'll debug eventually. Know what's inside before depending on it. Security: regulated environments (healthcare, finance) often require custom pipelines — every line must be auditable.

Four scenarios, four answers

1
Fast prototype / MVPUse LangChain or LlamaIndex. Build the RAG or agent quickly.
2
Production RAGUse LlamaIndex for retrieval + light custom orchestration. Best of both.
3
Latency-critical pathSkip framework, direct SDK calls. Every layer adds ms.
4
Security-audited pipelineCustom only. You need every line traceable and auditable.
Haystack (deepset): more production-focused than LangChain, YAML-defined pipelines, better for enterprise NLP. Instructor: not a full framework — just wraps any LLM to enforce Pydantic schema on output. Often all you need. Honest take: most experienced teams end up writing custom orchestration for their critical path after outgrowing frameworks. Rule of thumb: start with a framework, migrate the hot path to custom code once you understand the requirements.

Question 21  ·  Say it

Say it

"I treat LangChain and LlamaIndex as useful accelerators when I need fast RAG or agent flows, because they give me common patterns out of the box. But if the use case is latency-sensitive, security-critical, or very custom, I'll often write a smaller custom implementation instead of taking on the framework's complexity. The real question isn't 'framework or not' — it's 'where do I draw the boundary?'"

Tip: mention Instructor as a lighter-weight alternative when schema enforcement is the only need.
