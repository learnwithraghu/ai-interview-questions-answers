Question 20  ·  AI Engineer Interview Bootcamp

Managed API vs Open-Weight

What are the main differences between using a managed LLM API (like OpenAI GPT-4 or Anthropic Claude) versus accessing an open-weight model (like Llama 3) hosted on a cloud provider like Groq or Together AI?

Think in terms of control, cost, and who owns the ops burden.

Question 20  ·  The spectrum of control

Two camps, different trade-offs

Managed API
OpenAI  ·  Anthropic  ·  Google
  • Zero ops burden
  • Pay per token
  • No customisation
You focus on product
Open-Weight Hosted
Groq  ·  Together AI  ·  Replicate
  • You pick the model
  • Lower cost at scale
  • Fine-tuning possible
GPU is their problem
Managed API: provider handles infrastructure, scaling, and safety filtering. You focus on product. Open-weight hosted: you choose the model variant (Llama 3 8B, 70B, etc.) but GPU is someone else's problem. Cost crossover: managed API wins at low volume; open-weight hosted wins at high volume due to per-token pricing. Vendor lock-in: managed APIs have non-standard interfaces. Open-weight models use OpenAI-compatible APIs via providers.

Question 20  ·  The decision framework

Four factors to decide

1
Privacy & data residencyDoes your data leave your infrastructure? Managed APIs = yes.
2
CustomisationFine-tune on domain data? Only possible with open-weight.
3
Ops maturityDo you have ML infrastructure to manage? If not, start managed.
4
Cost at scaleMillions of calls/day makes hosted open-weight significantly cheaper.
Migration path: start on managed API to ship fast, migrate to hosted open-weight as volume grows. Groq: custom LPU chips running Llama/Mixtral at 500–800 tokens/sec — fastest inference available. Self-hosting (vLLM, Ollama): maximum control, lowest cost, highest ops burden. Fine-tuning advantage: a fine-tuned Llama 3 8B can outperform GPT-4o on specific domain tasks.

Question 20  ·  Say it

Say it

"In my experience, a managed API is best when you want low operational overhead, fast iteration, and a trusted provider handling scaling and updates. An open-weight model on hosted hardware is better when you need more control over the model, custom fine-tuning, and lower cost at scale — but you also need to invest in deployment and governance. I usually start teams on a managed API and migrate specific high-volume workloads to hosted open-weight as usage grows."

Tip: name-drop Groq (speed) and Together AI (variety) to show awareness of the landscape.
