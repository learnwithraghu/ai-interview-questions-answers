Question 25  ·  AI Engineer Interview Bootcamp

Multi-Agent Workflow Design

You need to build a complex, multi-agent workflow where a 'researcher' agent gathers data and a 'writer' agent drafts a report. How do you manage state, memory, and routing using frameworks like LangGraph or AutoGen?

Separate concerns: one agent gathers evidence, another turns it into prose.

Shared state: a typed Python dict or dataclass that flows through the graph — each agent reads from and writes to it. Separation of concerns: researcher focuses on recall; writer focuses on prose. Mixing them degrades both. LangGraph: models the workflow as a directed graph. Nodes are agents, edges are transitions, state flows between nodes. AutoGen: conversation-based — agents communicate via messages. State is implicit in the message history.

Question 25  ·  Framework comparison

Four frameworks to know

1
LangGraphTyped graph state, conditional edges, supports cycles. Best for complex branching flows.
2
AutoGen (Microsoft)Agents communicate via messages. GroupChatManager handles turn routing.
3
CrewAIRoles + tasks metaphor. Simple sequential/parallel orchestration. Great for quick prototypes.
4
Direct SDKCustom Python orchestrator calling Claude/GPT with different system prompts. Maximum transparency.
LangGraph cycles: the same agent can run multiple times in a loop until a condition is met — great for iterative research. CrewAI fastest start: define researcher + writer roles, assign tasks, a Crew runs them. Works in ~20 lines of code. AutoGen ALWAYS/NEVER/TERMINATE: configure human input mode per agent — powerful for controllable HITL workflows. Temporal for durability: if workflows run for hours and need to survive restarts, wrap your agents in Temporal activities.

Question 25  ·  Say it

Say it

"I'd separate the flow into a researcher agent that gathers and structures evidence, and a writer agent that consumes that evidence to draft the report. I'd use a workflow engine or graph framework to keep shared state in a central memory object and route the next step explicitly, so each agent stays focused and we can trace the full end-to-end process. For simple two-agent pipelines, CrewAI or direct SDK calls are sufficient; I'd upgrade to LangGraph when I need conditional branching or loops."

Tip: mention Temporal for durability if the interviewer asks about long-running or fault-tolerant workflows.
