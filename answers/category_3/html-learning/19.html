AI Engineer Interview — Q19

Vector DB Graceful Degradation

"Your vector database goes down, breaking the semantic search feature of your application. How do you design a highly available GenAI architecture that gracefully degrades when retrieval fails?"

💡 When one component fails, the system should keep working at reduced capacity — not crash.

Circuit breaker: Monitors health endpoint. Detects failure in 5s and stops routing before users notice. Read replicas: Pinecone/Weaviate managed tiers offer multi-region replicas — failover at the infrastructure level. BM25/Elasticsearch fallback: Keyword search is less semantic but always available and genuinely useful. Transparency: Show users a banner — "AI search is in basic mode" — silent degradation erodes trust.

AI Engineer Interview — Q19 — Degradation Hierarchy

Four-level fallback chain

1
Primary vector DB — Full semantic search, best results.
2
Keyword search (BM25/Elasticsearch) — Less semantic, still genuinely useful.
3
Redis cached results — Pre-populated for most common queries.
4
LLM-only mode — No RAG context, model uses training data only. Show clear warning to user.
Cache common queries: Pre-populate Redis with results for your top-50 most-asked questions. LLM-only as last resort: Keeps the product functional, but tell users — never silently serve degraded answers. Recovery: Circuit auto-closes and routes back to primary once health check passes after cooldown. Test your fallbacks: Run degradation drills — it's the only way to know each level actually works.

AI Engineer Interview — Q19 — Say It

Say it

"The key principle is graceful degradation — the app should keep working at reduced capacity, not crash entirely. I'd achieve this with a circuit breaker that detects vector DB failures within seconds and switches to a fallback strategy automatically. My primary fallback would be keyword search against the same corpus. I'd also cache results for common queries in Redis. Throughout all of this, I'd show users a clear banner so they know retrieval quality is temporarily reduced."

Circuit breaker → BM25 keyword search → Redis cache → LLM-only + user banner.
