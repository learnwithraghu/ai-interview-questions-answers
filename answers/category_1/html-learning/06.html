Interview question

In your AI coding experience, what is an example of an LLM hallucination you saw, and why do you think it happened?

The answer should describe a real coding-shaped falsehood.

A coding hallucination often looks perfectly reasonable: a fake method name, file path, config option, or library API. The problem is not syntax; the claim is ungrounded. The code can look natural while still being false in the actual repo. That is why believable code still needs verification against real project evidence.

Visual explanation

Grounding gaps

Missing repo context
Stale API memory
Vague request

Hallucinations happen when the model predicts code-shaped tokens without enough reality checks. Missing repo context, stale framework knowledge, or an underspecified prompt can push it toward plausible guesses. Reading repo files, dependency versions, and documentation turns guessing into grounded work. The key interview move is explaining why the falsehood happened.

Answer

Say it

"Yes. One common example I have seen is an AI coding assistant inventing a helper function or library method that looked completely reasonable but did not exist in the actual repo."
