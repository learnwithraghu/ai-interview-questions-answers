Interview question

What is the purpose of AGENTS.md in a GitHub repository?

Answer it as repo-level context for AI coding agents.

AGENTS.md works like onboarding docs for an AI assistant. It tells the agent what the repo is, which conventions matter, what files must stay in sync, and where deeper workflow instructions live before the agent starts editing. The file becomes the repo-level entry point for consistent AI behavior across tools.

Visual explanation

Repeatable context

Blank session
Read repo rules
Consistent work

The value is repeatability. Instead of re-explaining project rules in every chat, teams version-control those rules once. Future agent sessions start with shared context, follow the same conventions, and avoid rediscovering basic project workflow every time. That makes agent output easier to review, maintain, and trust across repeated changes later.

Answer

Say it

"AGENTS.md is a markdown file at the repo root that gives AI coding agents durable project context: conventions, structure, and rules they should follow."
