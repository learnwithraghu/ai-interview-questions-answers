What is the difference between input tokens and output tokens in LLM APIs?

This is about cost, limits, and what actually enters the request.

Visual explanation

Input tokens are everything packed into the request: system prompt, chat history, RAG context, tool definitions, and the latest user message. Output tokens are the generated completion. If it goes into the payload, it is input; if the model writes it back, it is output. This is why the latest user text is only part of the bill.

Visual explanation

Both are metered

prompt_tokenswhat you send
completion_tokenswhat it writes

Both sides count toward the context window, and providers usually price them separately. Long chat histories grow the input side every turn, leaving less room for the answer. Chatty responses increase output cost, so production systems track both counts and cap generation. Monitoring only prompt size misses half of the cost story.

Answer

Say it

"Input tokens are everything you send to the model. Output tokens are what the model generates back. Both count toward the context window limit."
