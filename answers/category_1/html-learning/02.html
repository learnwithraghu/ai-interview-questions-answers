Question 02

Interview question

Explain the concept of few-shot prompting and how it differs from zero-shot prompting. Provide an example of when you would use it.

Look for the difference between instruction and demonstration.

Visual explanation

Ask cold vs show pattern

ZERO-SHOT
->
task only
FEW-SHOT
->
examples + task
Zero-shot: task arrives cold from instructions alone.Model burden: infer format, tone, and boundary.Few-shot: input-output pairs go on the board first.Examples teach: format, edge cases, and domain vocabulary.No training: the pattern is learned inside the prompt.Effect: the model copies the visible pattern instead of inventing its own format.

Visual explanation

When examples help

Simple taskzero-shot
Odd formatfew-shot
Domain labelsfew-shot
Use zero-shot: common task, simple output, clear instruction.Use few-shot: output drifts or format is unusual.Domain labels: examples anchor taxonomy and edge cases.Not RAG: these are demonstrations, not background documents.Token cost: every example is resent on each request.Production note: examples improve consistency but increase request size and latency.

Answer

Say it

"Zero-shot means I describe the task with no examples. Few-shot means I include a few input-output pairs to show the model exactly what pattern to follow."
