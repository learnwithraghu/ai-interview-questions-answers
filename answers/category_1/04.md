# Question 4: Low-Rank Adaptation (LoRA)

**Question:** What is the purpose of Low-Rank Adaptation (LoRA) in fine-tuning?

**Answer:**
Low-Rank Adaptation (LoRA) is a Parameter-Efficient Fine-Tuning (PEFT) technique designed to make fine-tuning large language models computationally feasible and memory-efficient.

**The Purpose and How it Works:**
Normally, full fine-tuning requires updating all the original weights of a massive model (which could be billions of parameters), demanding huge amounts of GPU memory and compute.

Instead of modifying the original weights, LoRA freezes the original pre-trained model weights. It then injects small, trainable "rank decomposition matrices" into the layers of the Transformer architecture. 

During training, only these small, low-rank matrices are updated. 

**Key Benefits:**
1. **Dramatically Reduced Parameters:** It reduces the number of trainable parameters by up to 10,000x, and the GPU memory requirement by a factor of 3x.
2. **Cost-Effective:** You can fine-tune massive models on consumer-grade GPUs or single GPUs rather than needing expensive clusters.
3. **Modular and Swappable:** The trained LoRA weights (adapters) are extremely small files (a few megabytes). You can have one base model loaded in memory and hot-swap different LoRA adapters for different tasks on the fly, without needing multiple copies of the full model.
4. **Preserves Base Performance:** Because the original weights are frozen, LoRA helps mitigate "catastrophic forgetting," where a model loses its general knowledge while learning a specific task.
