# Question 3: Dense Retrieval in RAG

**Question:** How does dense retrieval work in RAG pipelines, and what makes it more effective than keyword-based search for semantic queries?

**Answer:**
**How Dense Retrieval Works:**
1. **Embedding:** Both the corpus of documents (during ingestion) and the user's query (at runtime) are passed through an embedding model (like text-embedding-ada-002). This converts the text into dense, high-dimensional mathematical vectors.
2. **Vector Database:** The document embeddings are stored in a Vector Database (like Pinecone, Weaviate, or Milvus).
3. **Similarity Search:** When a query vector is generated, the vector database calculates the mathematical distance (usually Cosine Similarity or Dot Product) between the query vector and all document vectors.
4. **Retrieval:** The documents with vectors closest to the query vector are retrieved as the context.

**Why it is more effective than keyword-based search (Sparse Retrieval/BM25):**
Keyword search relies on exact word matches or overlapping terms. If a user searches for "automobile" but the document uses the word "car," a keyword search will likely fail.

Dense retrieval captures the **semantic meaning** and context of the text. Because "automobile" and "car" are conceptually similar, their vectors will be close together in the vector space. Therefore, dense retrieval can find highly relevant information even if the exact keywords are missing from the text, making it much better at handling natural language questions, synonyms, and nuanced queries.
