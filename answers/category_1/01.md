# Question 1: System, User, and Assistant Prompts

**Question:** What is the difference between a system prompt, a user prompt, and an assistant response in modern chat-based LLM APIs?

**Answer:**
In modern chat-based LLM APIs (like OpenAI's Chat Completions API), interactions are structured as a conversation with specific roles:

1. **System Prompt (System Message):**
   - **Purpose:** Sets the overarching behavior, persona, context, and constraints for the LLM. It acts as the "instructions" for how the model should behave throughout the conversation.
   - **Example:** "You are a helpful, senior Python engineer. Always provide code with comments and explain your reasoning."

2. **User Prompt (User Message):**
   - **Purpose:** This is the actual input, query, or instruction from the end-user. It represents what the user wants the model to do or answer at that specific moment.
   - **Example:** "How do I reverse a string in Python?"

3. **Assistant Response (Assistant Message):**
   - **Purpose:** This is the output generated by the LLM. Including past assistant responses in the API call allows the model to maintain context of the ongoing conversation (conversational memory).
   - **Example:** "You can reverse a string in Python using slicing, like this: `reversed_string = my_string[::-1]`."

By separating these roles, developers have much finer control over the model's behavior and can prevent user inputs from overriding core instructions (though prompt injection is still a risk to manage).
