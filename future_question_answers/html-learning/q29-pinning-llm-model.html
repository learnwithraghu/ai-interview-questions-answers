Question 29

An AI Engineer deployed a RAG system and it functioned fine, but after 2 months it started giving bad results. No code change and no data change occurred. What could be the issue?

The Problem

Why Systems Drift: Dynamic Aliases

App Calls Alias
model="gpt-4o"
→
LLM Gateway
(API Resolves Pointer)
→
Month 1: gpt-4o-2024-05-13
Month 2: gpt-4o-2024-08-06 (Drift!)
Dynamic Aliases Generic model IDs like gpt-4o automatically point to whichever version the provider currently deems standard.
Hidden Upgrades API providers periodically update these alias pointers under the hood to new model snapshots without notifying you.
Behavioral Shifts Even minor model updates change prompt sensitivity, formatting rules, and reasoning styles, causing systems to fail.

The Solution

The Fix: Immutable Version Pinning

App Pins ID
model="gpt-4o-2024-05-13"
→
LLM Gateway
(Fetches Pinned Model)
→
Month 1 & 2: gpt-4o-2024-05-13
(Guaranteed Static Behavior)
Dated Snapshots Using specific IDs like gpt-4o-2024-08-06 ensures you interact with a locked, unchanging model binary.
Deterministic Performance Your outputs remain consistent and stable over time because the underlying neural weights never shift.
Controlled Migrations Upgrades are done explicitly by running offline evaluations on golden datasets before updating version pins.

Interview Response

How to Answer in an Interview

"If a production RAG system suddenly starts giving bad results with no code or data changes, the first thing I would check is whether we pinned our model versions. Using dynamic aliases like gpt-4o is a major risk because API providers periodically update these aliases to point to newer underlying snapshots. These updates can alter prompt alignment, change output formats, or cause regressions on specific tasks. To prevent this, I always pin model calls to specific, immutable, dated snapshots in production. When we want to upgrade to a newer model version, we treat it as a database migration: we run the new model against a golden evaluation dataset to measure performance and tune our prompts before making the switch in production."
